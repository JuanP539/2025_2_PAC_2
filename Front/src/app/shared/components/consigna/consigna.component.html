Consigna — Ejercicio Reflection

Objetivo

Aplicando las buenas prácticas vistas en el curso, implementar una funcionalidad que permita listar, desde el Front, los nombres de las DLL que contienen al menos un tipo público y concreto que implemente ImporterInterface.

Backend (.NET)

  1. Implementar el endpoint GET /api/reflection/importers.
  2. El endpoint debe escanear las DLL dentro del directorio reflection.
  3. Por cada ensamblado cargable, verificar si contiene al menos un tipo público y no abstracto que implemente ImporterInterface.
  4. Devolver HTTP 200 con un arreglo de strings (string[]) que contenga los nombres de las DLL encontradas.
  5. Si no se encuentra ninguna DLL, devolver HTTP 200 con una lista vacía.

Frontend (Angular)

  1. Agregar una ruta nueva /reflection.
  2. Agregar en el navbar un ítem que navegue a esa ruta.
  3. Cada vez que se accede a la ruta /reflection, incrementar un contador en memoria.
  4. Configurar una guarda (guard) asociada a la ruta /reflection que verifique el valor del contador:
    • Si el contador es mayor a 20, el acceso a la ruta debe bloquearse.
  5. En la página /reflection, incluir un botón que invoque el endpoint del backend.
  6. Mostrar los nombres devueltos por el endpoint en una lista.
  7. Mostrar los estados de carga, vacío y error.
